# 03A Test Answers

> 작성자: Main Lead (교육생 답변 대리 기록)
> 참고: 아래 답변은 `03_test_questions_and_plan.md`의 2번 섹션 질문 5개에 대한 교육생의 최종 답변이다.
> Tester와 Builder는 이 답변을 기준으로 각자의 계획을 확정/조정해야 한다.

## 질문 1 — 빈 입력 처리

**답변: 빈 입력(빈 문자열 또는 공백만 있는 입력)은 저장하지 않는다.**

- 빈/공백 입력은 분류 로직을 실행하거나 결과를 저장하지 않고, 안내만 표시한다.

## 질문 2 — 우선순위 P0/P1/P2 분류 방식

**답변: 우선순위는 자동 규칙 분류가 아니라 사용자가 직접 선택하는 수동 방식으로 한다.**

- 키워드 기반 자동 분류 로직은 사용하지 않는다.
- 입력 화면에 P0/P1/P2 선택 UI(예: 라디오 버튼/드롭다운)를 두고 사용자가 직접 고른다.

## 질문 3 — 리뷰 필요 여부 판정 기준

**답변: 리뷰 필요 여부는 체크박스로 사용자가 직접 입력한다.**

- 우선순위나 키워드로 자동 판정하지 않는다.
- 사용자가 체크박스를 선택하면 "리뷰 필요", 선택하지 않으면 "리뷰 불필요"로 처리한다.

## 질문 4 — 담당 역할 / 다음 액션 형식

**답변: 다음 액션은 고정 옵션으로 한다.**

- 자유 텍스트가 아니라 사전에 정의된 고정 목록 중에서 선택하는 방식으로 구현한다.
- 담당 역할 역시 동일하게 고정 목록 방식을 따른다 (예: Frontend, Backend, QA, Infra 등).

## 질문 5 — localStorage 저장 범위 포함 여부

**답변: localStorage 저장 범위를 포함한다.**

- 새로고침 후에도 이전 입력 목록(우선순위/리뷰 필요 여부/담당 역할/다음 액션 포함)을 다시 볼 수 있어야 한다.
- Builder와 Tester는 저장/조회 로직과 관련 테스트를 계획에 포함해야 한다.

## 요약 (Tester/Builder 반영용)

| 항목 | 결정 |
|---|---|
| 빈 입력 | 저장하지 않음 (안내만 표시) |
| 우선순위 분류 | 수동 선택 (자동 규칙 미사용) |
| 리뷰 필요 여부 | 체크박스 수동 입력 |
| 역할/다음 액션 | 고정 목록 선택 |
| localStorage 저장 | 포함 (새로고침 후에도 이전 입력 목록 유지) |
